# PR Review Checklist

## Correctness and scope

- [ ] Change implements approved spec only.
- [ ] No out-of-scope behavior introduced.

## Quality and safety

- [ ] Tests cover expected and edge behavior.
- [ ] Security and NFR constraints respected.
- [ ] No obvious regressions introduced.

## Traceability and context

- [ ] Requirement and spec links included.
- [ ] Feature ID present in PR and artefacts.
- [ ] Context updates identified or completed.
