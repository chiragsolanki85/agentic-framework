## Baseline expectations

- Readability and maintainability first
- Small, atomic changes
- Tests added/updated for behaviour changes
- Security and dependency hygiene maintained

## Practical interpretation

- Prefer incremental merges over large rewrites.
- Keep implementation aligned to approved spec.
- Use explicit naming and predictable file boundaries.
- Add validation evidence to implementation logs.
