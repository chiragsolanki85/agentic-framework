## Purpose

Transform discovery context into refined, BDD-ready feature requirements with explicit Given/When/Then acceptance criteria. Requirements must be traceable to foundations and approved before specification begins.

[Explore this stage in the SDLC Flow](/flow#requirements)

## Inputs

Foundations, constraints, initiative intent, and discovery notes.

## Outputs

`FEATURE-<EPIC>-<NNN>.md` with refined-ready checklist evidence.

## Prompt to run

- `framework/prompts/02-requirements-agent.md`
- Gate: `framework/prompts/03-refined-ready-gate-review.md`

## Human gate

**Owner:** Product Lead  
**Decision:** Approve refined-ready gate before specification.

- Reads: `context/00_foundations/**/*.md`
