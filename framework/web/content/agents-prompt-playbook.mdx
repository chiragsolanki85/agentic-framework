## Standard execution sequence

Use this order for one feature (`FEAT-<EPIC>-<NNN>`):

1. `framework/prompts/00-session-bootstrap.md`
2. `framework/prompts/02-requirements-agent.md`
3. `framework/prompts/03-refined-ready-gate-review.md`
4. `framework/prompts/04-spec-agent.md`
5. `framework/prompts/05-spec-approval-gate.md`
6. `framework/prompts/06-implementation-agent.md`
7. `framework/prompts/07-review-agent.md`
8. `framework/prompts/11-validation-agent.md`
9. `framework/prompts/12-deployment-agent.md`
10. `framework/prompts/13-feedback-agent.md`
11. `framework/prompts/14-context-lifecycle-agent.md`

Always run:

- `framework/prompts/09-handoff-log-entry.md` after each gate
- `framework/prompts/10-orchestrator-whats-next.md` if blocked or uncertain

## Governance prompts (when to use)

- `framework/prompts/governance/G1-governance-compliance-audit.md` for baseline governance checks
- `framework/prompts/governance/G2-traceability-audit.md` for requirement-to-release traceability
- `framework/prompts/governance/G3-human-gate-readiness.md` before major approvals
- `framework/prompts/governance/G4-risk-escalation-check.md` when risk signals increase
- `framework/prompts/governance/G5-v2-gate-audit.md` for end-to-end lifecycle closure
