## Purpose

Execute the approved specification in atomic, verifiable code changes with traceable commits and implementation evidence. Implementation follows the spec — deviations require spec revision and re-approval.

[Explore this stage in the SDLC Flow](/flow#implementation)

## Inputs

Approved `SPEC-FEAT-<FEATURE-ID>.md`, codebase, and coding standards.

## Outputs

Code changes, tests, and `IMPLEMENTATION-FEAT-<FEATURE-ID>.md`.

## Prompt to run

`framework/prompts/06-implementation-agent.md`

## Human gate

**Owner:** Engineering Lead (via Review Agent)  
**Decision:** PR review and merge approval before validation.
