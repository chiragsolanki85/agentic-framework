## Happy-path agents

Specialized agents with bounded responsibilities — see the [SDLC Flow](/flow) for the interactive handoff diagram.

- Requirements Agent
- Spec Agent
- Implementation Agent
- Review Agent
- Validation Agent
- Deployment Agent
- Feedback Agent
- Context Lifecycle Agent

## Supplementary agents

- **Context Manager Agent** — on-demand context alignment during delivery (prompt 08). Use when shared docs need updating mid-flight; not part of the standard gate sequence.

## Why specialized agents

- Prevents role drift and uncontrolled autonomy
- Improves quality through clear IO contracts
- Makes escalation and approval ownership explicit

## Agent responsibilities (at a glance)

| Agent | Primary input | Primary output | Human gate | Path |
|---|---|---|---|---|
| Requirements | Foundations and constraints | Feature BDD requirement | Refined-ready approval | Happy-path |
| Spec | Approved requirement | Technical spec | Spec approval | Happy-path |
| Implementation | Approved spec | Code and implementation log | PR/merge approval | Happy-path |
| Review | Requirement/spec/implementation | Risk and quality findings | Engineering approval decision | Happy-path |
| Validation | Requirement/spec/implementation | Validation report | Suite approval | Happy-path |
| Deployment | Validation and readiness | Deployment report | Go/no-go + verification | Happy-path |
| Feedback | Telemetry and outcomes | Outcome review | Product triage decision | Happy-path |
| Context Lifecycle | All stage outputs | Diff/consistency closure evidence | Context complete verification | Happy-path |
| Context Manager | Merged PR and changed code | Updated shared context docs | Context completeness check | On-demand |
