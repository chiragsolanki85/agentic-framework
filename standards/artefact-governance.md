# Artefact Governance Rules

## Rule 1: Artefacts are source of truth
All lifecycle decisions must be represented in version-controlled markdown artefacts.

## Rule 2: No build without approved context
No coding begins before requirement and spec gates are approved.

## Rule 3: Explicit handoffs
Every stage transition must be logged in `logs/handoffs/30_handoffs.md`.

## Rule 4: Context completeness
A feature is not done until code and context updates are complete and approved.
