# Traceability Policy

## Feature ID policy

Each feature must use one stable ID: `FEAT-<EPIC>-<NNN>`.

## Required links

- Requirement file: `01_requirements/features/FEATURE-<EPIC>-<NNN>.md`
- Spec file: `02_specs/SPEC-FEAT-<EPIC>-<NNN>.md`
- Implementation log: `03_implementation/IMPLEMENTATION-FEAT-<EPIC>-<NNN>.md`
- PR description must include feature ID and links.

## Commit/PR policy

- Commit messages should include the feature ID when applicable.
- PR template/checklist must confirm requirement/spec references.
